================================================================================
                          ACTIVE MENU DETECTION
================================================================================

HOW IT WORKS:

Blade Template:
  <a href="{{ route('admin.gejala.index') }}"
     class="@if(request()->routeIs('admin.gejala.*')) active @endif">

LOGIC:
  1. Check current route with request()->routeIs()
  2. Use wildcard (*) for pattern matching
  3. Apply 'active' class if matches
  4. CSS applies purple highlight

EXAMPLES:
  - On /admin/gejala → 'Kelola Gejala' highlighted
  - On /admin/gejala/create → 'Kelola Gejala' highlighted
  - On /admin/gejala/{id}/edit → 'Kelola Gejala' highlighted
  - On /admin/penyakit → 'Kelola Penyakit' highlighted

PATTERN:
  route('admin.gejala.index')      matches  request()->routeIs('admin.gejala.*')
  route('admin.gejala.create')     matches  request()->routeIs('admin.gejala.*')
  route('admin.gejala.store')      matches  request()->routeIs('admin.gejala.*')
  route('admin.gejala.edit')       matches  request()->routeIs('admin.gejala.*')
  route('admin.gejala.update')     matches  request()->routeIs('admin.gejala.*')
  route('admin.gejala.destroy')    matches  request()->routeIs('admin.gejala.*')

================================================================================
                         TROUBLESHOOTING
================================================================================

ISSUE: Sidebar not appearing
  SOLUTION: 
    1. Check if logged in as admin
    2. Verify URL contains /admin/*
    3. Clear browser cache
    4. Check browser console for errors

ISSUE: Menu items not working
  SOLUTION:
    1. Verify routes in web.php
    2. Check route names match sidebar
    3. Verify auth middleware
    4. Check database connectivity

ISSUE: Mobile toggle not working
  SOLUTION:
    1. Check JavaScript console for errors
    2. Verify Bootstrap JS is loaded
    3. Check viewport meta tag
    4. Try different mobile browser

ISSUE: Styling looks wrong
  SOLUTION:
    1. Clear browser cache
    2. Hard refresh (Ctrl+Shift+R)
    3. Check browser DevTools
    4. Verify no custom CSS conflicts

ISSUE: Performance issues
  SOLUTION:
    1. Check network tab in DevTools
    2. Verify database is responsive
    3. Check server load
    4. Optimize database queries
